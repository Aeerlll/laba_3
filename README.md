## 🚀 Как запустить

1. Установите [.NET 8.0 SDK](https://dotnet.microsoft.com/download)
2. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/Aeerlll/laba_3.git
   ```
3. Откройте решение `laba_3.sln` в Visual Studio 2022 или Rider
4. Восстановите пакеты: `dotnet restore`
5. Запустите проект: `Ctrl+F5` или `dotnet run`

## 📈 Что умеет приложение

- Загружать данные из CSV-файлов с проверкой формата и обработкой ошибок
- Отображать статистику в удобной таблице с сортировкой

## 🌿 Как мы работали с Git

Мы использовали упрощённую модель Git Flow:

- `main` — стабильная версия, готовая к сдаче
- `dev` — ветка для интеграции готовых функций
- `feature/weather-forecast` — разработка модуля температуры
- `feature/expenses-analyzer` — разработка модуля расходов
- `feature/migration-module` — разработка модуля миграции
- `feature/population-variant-5` — разработка модуля численности населения

Каждый участник работал в своей ветке, периодически пушил изменения и создавал Pull Request в `dev`. Перед слиянием код обязательно проходил ревью: коллеги оставляли комментарии, предлагали улучшения, указывали на потенциальные проблемы. Такой подход помог не только избежать багов, но и научиться писать более чистый и поддерживаемый код.

## 📐 Как работает прогноз

Мы используем метод экстраполяции по скользящей средней. Пользователь задаёт параметр `N` — размер окна усреднения. Прогноз на следующий период рассчитывается как среднее арифметическое последних `N` значений. При расчёте каждого следующего шага прогнозируемое значение добавляется в окно, что позволяет строить цепочку прогнозов.

Метод хорошо сглаживает случайные колебания и помогает увидеть основную тенденцию, что особенно полезно при работе с реальной статистикой, где данные часто «шумные».
